먼저 다음 문서를 모두 읽어줘: CLAUDE.md, docs/PROJECT_CONTEXT.md, docs/PRD.md,
docs/DECISIONS.md, docs/ARCHITECTURE.md.

읽었으면, 코드를 짜기 전에 다음을 말로 먼저 해줘:
1) 네가 이해한 이 프로젝트의 한 줄 요약
2) DECISIONS.md의 신뢰 불변식 6가지 중 코드에서 가장 강제하기 어려운 것과 그 이유
3) 첫 번째 작업(스캐폴딩)에서 만들 폴더·파일 계획과, 더 단순한 대안이 있다면 그것

거기까지는 코드를 짜지 마. 내가 계획에 동의하면 그때 시작한다.

작업 원칙:
- 한 번에 한 슬라이스만. 끝나면 멈추고 내가 테스트하게 해줘.
- 짜기 전에 항상 "무엇을·왜·위험·더 단순한 대안"을 먼저 설명해.
- 신뢰 불변식(특히 출처 필수 / 확인 게이트 단일 쓰기 / 무근거 답변 금지)을 절대 어기지 마.
- API 키·비밀은 코드나 git에 절대 넣지 말고 .env로만.

빌드 순서는 ARCHITECTURE.md의 "빌드 순서"를 따르되, 1단계 스캐폴딩 계획부터 보여줘.
